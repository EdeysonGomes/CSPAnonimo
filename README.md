# Supplementary Materials for the Competency Specification Process (CSP)

## Overview

This repository contains supplementary materials associated with the application and evaluation of the **Competency Specification Process (CSP)** in Computing Education contexts.

The artifacts document how educational tasks were analyzed and transformed into competency specifications through iterative cycles of elicitation, review, refinement, consolidation, reuse, and specialization. Together, these materials provide traceable evidence of the methodological procedures described in the accompanying study.

The repository is intended to support transparency, reproducibility, and secondary analysis of the CSP, enabling readers to examine the evolution of competency specifications across multiple application cycles.



## Anonymous Review Notice

This repository has been prepared to support anonymous academic review.

All materials have been reviewed to remove information that could directly identify the authors or affiliated institutions. Nevertheless, reviewers are encouraged to disregard any residual metadata that may inadvertently reveal authorship.

The repository should not contain:

* author names;
* institutional affiliations;
* email addresses;
* acknowledgments;
* project identifiers;
* personal references;
* version-control history;
* file metadata containing authorship information.



## Repository Purpose

The purpose of this repository is to provide empirical and methodological evidence regarding:

* the use of educational tasks as sources for competency elicitation;
* the specification of competencies through Knowledge–Skill–Disposition (K–S–D) structures;
* the iterative review and refinement of competency specifications;
* decisions involving competency reuse, specialization, consolidation, and exclusion;
* the evolution of competency models across successive CSP cycles;
* the methodological analysis reported in the associated research study.

This repository is **not intended to evaluate student performance, learning outcomes, or instructional effectiveness**. Its primary focus is the process of competency specification and refinement.



## Repository Structure

| Folder                | Description                                                                                                                      |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `CSP.cycles/Cycle01/` | Initial competency elicitation cycle based on educational task analysis.                                                         |
| `CSP.cycles/Cycle02/` | Review and refinement cycle, including competency reuse and restructuring activities.                                            |

## Artifact Types

The repository contains the following artifact categories:

| Artifact      | Description                                                                                  |
| ------------- | -------------------------------------------------------------------------------------------- |
| `Description` | Educational task description used as input for competency elicitation.                       |
| `CSP-Report`  | Report documenting competency specification activities and resulting competency definitions. |
| `CSRP-Report` | Report documenting the competency review process and expert evaluation findings.             |
| `Adjustments` | Record of revisions implemented following the review process.                                |
| `Competency`	| Specification of a competency (atomic, composite, or specialized), stored as CXX.md.         |
| `Lifecycle`	| Historical record of a competency’s evolution, stored as CXX_LifeCycle.md.                   |



## Recommended Reading Sequence

For each educational task, the recommended reading order is:

1. Educational task description;
2. CSP specification report (`CSP-Report`);
3. Competency review report (`CSRP-Report`), when available;
4. Adjustment report (`Adjustments`), when available.

This sequence allows readers to follow the complete lifecycle of a competency specification, from task analysis through review and refinement.



## Relationship to the Associated Study

The artifacts contained in this repository complement the accompanying research by providing detailed evidence of:

* competency elicitation from educational tasks in Computing Education;
* iterative refinement through structured expert review;
* competency reuse and specialization across tasks;
* evolution of competency specifications throughout successive CSP applications;
* methodological decisions that informed the final competency catalog.

The repository should be interpreted as supporting material for understanding the CSP methodology and its application rather than as a standalone educational intervention dataset.



## Ethical Considerations

The materials contained in this repository consist exclusively of instructional and methodological artifacts produced during competency specification activities.

The repository does not include:

* student submissions;
* assessment records;
* grades;
* learning analytics data;
* personal information;
* identifiable participant data.

Any information that could potentially identify individuals or institutions should be removed or anonymized prior to public release.



## Citation and Reuse
