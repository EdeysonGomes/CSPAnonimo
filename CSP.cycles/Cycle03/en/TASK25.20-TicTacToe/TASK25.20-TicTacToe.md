# TASK25.2 – Tic-Tac-Toe Implementation


## Instructional Context

This task is proposed in the **Programming Logic** course of the **[TECHNICAL_PROGRAM] (1st year)** at **[INSTITUTION] – [CAMPUS]**. It is designed to consolidate fundamental concepts of **structured programming** through the implementation of a well-defined computational problem.

The activity is carried out in **small groups (2–3 students)**, with **individual accountability ensured through an oral assessment**.



## Task Objective

The objective of this task is to enable students to **design, implement, and explain a complete interactive program**, applying basic programming constructs to model game logic, control program flow, and manage user interaction in a consistent and robust manner.



## Task Description

Students must implement a **text-based or graphical tic-tac-toe game (3×3 board)** that supports **two human players** alternating turns using the symbols **“X”** and **“O”**.

The program must:

* Represent the game board using appropriate data structures (e.g., lists or matrices).
* Control the game flow through loops and conditionals.
* Validate user inputs, rejecting invalid or illegal moves.
* Detect and report game termination conditions, including:

  * victory by rows, columns, or diagonals;
  * draw situations.
* Clearly display the current state of the board after each move.

The implementation must be **functionally complete**, **logically consistent**, and **robust against incorrect inputs**.



## Development Process

To complete the task, students will rely on:

* A programming environment (e.g., Python or another language defined by the instructor).
* Laboratory infrastructure with individual workstations.
* Prior instructional support covering:

  * variables and control structures;
  * lists and matrices;
  * loops and conditionals;
  * basic function definition and usage.

Collaboration is encouraged during development; however, **each student must demonstrate individual understanding of the solution**.



## Expected Evidence

The following evidence is collected to support assessment:

* **Source code** implementing the game logic.
* **Program execution**, demonstrating correct behavior during a full match.
* **Individual oral explanation**, in which each student:

  * describes the structure of the code;
  * explains the role of key functions and control structures;
  * traces the execution flow from initialization to game termination.



## Assessment Criteria

Evaluation considers three complementary dimensions:

1. **Technical Dimension**

   * The program runs without errors.
   * Game rules and move validations are correctly implemented.

2. **Cognitive Dimension**

   * The student correctly explains how the program is structured.
   * The student justifies the use of data structures, loops, and conditionals.
   * The student is able to trace and explain the execution flow of a complete match.

3. **Attitudinal Dimension**

   * The student demonstrates effective participation in the group work.
   * The student shows respect for peers and collaborative behavior.
   * Responsibilities are fairly distributed and acknowledged during the presentation.
