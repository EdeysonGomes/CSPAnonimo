# TASK25.23 – CSP-Report

## Introduction

This CSP-Report documents the competency specification of **TASK25.23 – Interactive Quiz: Fundamental Concepts of Hardware and Software**, an instructional task conducted with 1st-year students of the [TECHNICAL_PROGRAM] at [INSTITUTION].

The task uses a 15-question interactive quiz on [QUIZ_PLATFORM] to review and consolidate concepts from a theoretical block on computer architecture, including hardware components, software categories, basic system organization, and relationships between hardware, software, and performance. 



## 1. Instructional Entity Analysis

### Title

    Interactive Quiz – Fundamental Concepts of Hardware and Software

### Description

Learners participate in an interactive quiz composed of 15 multiple-choice questions about hardware, software, and basic computer architecture. The questions address identification and function of hardware components, differences between system software and application software, relations between hardware and performance, and problem-based scenarios involving real computational systems.  

After each question, the instructor conducts a guided discussion to explain the correct answer, clarify misunderstandings, and relate the topic to previous theoretical lessons.  



### Development Process

Learners are expected to:

1. read and interpret quiz questions;
2. identify relevant hardware or software concepts;
3. select the most appropriate answer under time constraints;
4. compare their answer with the immediate feedback;
5. participate in guided discussion;
6. explain or revise their reasoning when necessary.



### Expected Evidence

The task produces four main types of evidence:

1. **Quiz performance records**, including correct answers and response time.
2. **Responses to scenario-based questions**, showing conceptual application.
3. **Oral justifications during discussion**, showing reasoning and technical vocabulary.
4. **Participation in feedback cycles**, showing correction of misconceptions and engagement. 



### Acquisition Context

The task was implemented in the **Computer Architecture** unit of the **1st-year [TECHNICAL_PROGRAM] at [INSTITUTION]**, after completion of the theoretical block on computer architecture, hardware, software, and organization of computational systems. 

The activity took place in a computer laboratory using mobile phones and institutional computers with internet access.

## 3. Learning Objectives

By the end of TASK25.5, the learner should be able to:

1. **Identify** basic hardware components and describe their functions.
2. **Distinguish** system software from application software.
3. **Explain** the role of the operating system in managing computational resources.
4. **Relate** hardware and software elements in basic system operation.
5. **Select** appropriate answers based on conceptual reasoning.
6. **Justify** responses using technical vocabulary.
7. **Revise** misconceptions based on immediate feedback and discussion.

## 6. Evidence and Assessment Interpretation

TASK25.5 supports three evidential layers.

### 6.1 Artifact-Level Evidence

Quiz performance records generated by [QUIZ_PLATFORM] provide evidence of selected answers, accuracy, and response time.

### 6.2 Performance-Level Evidence

Scenario-based answers and participation in feedback cycles provide evidence of conceptual application, correction of misconceptions, and reasoning under time constraints.

### 6.3 Learner-Level Evidence

Oral justifications during guided discussion provide learner-specific evidence of understanding, vocabulary use, and reasoning.

### Analytical Note

This distinction is important because quiz scores alone should not be treated as sufficient evidence of competence. [Ontology] should represent the quiz score as one evidence source, while oral justification and feedback-based revision provide additional evidence for interpreting learner competence.
