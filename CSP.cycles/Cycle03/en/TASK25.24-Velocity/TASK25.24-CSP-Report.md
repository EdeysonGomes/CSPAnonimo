# TASK26.6 – CSP-Report

## Introduction

This CSP-Report documents the competency specification of **TASK26.6 – Program to Explore the Average Velocity Formula**, an interdisciplinary task integrating Physics and Computing in the context of an [TECHNICAL_PROGRAM].

The task requires learners to develop an **interactive program** that receives two of three variables — average velocity (Vₘ), distance (Δs), and time (Δt) — and computes the missing one using the appropriate rearrangement of the formula. 

From the perspective of [Ontology], the task is primarily relevant for **Computing competencies**, including:

- conditional logic,
- control flow,
- input validation,
- iterative execution,
- problem decomposition,
- and computational modeling.

The Physics knowledge involved (velocity formula and algebraic rearrangement) is treated as a **prerequisite knowledge base**, not as the focus of competency specification.



## 1. Instructional Entity Analysis

### Title

    Program to Explore the Average Velocity Formula

### Description

Learners develop an interactive program that:

- receives up to three values (Vₘ, Δs, Δt);
- identifies which value is missing;
- computes the missing value using the appropriate formula;
- validates inputs;
- allows repeated execution through a loop.

The program must ensure that exactly two valid values are provided and must handle invalid input cases appropriately. 



### Development Process

Learners are expected to:

1. interpret the problem as a computational task;
2. identify inputs, outputs, and constraints;
3. design logic to detect the missing variable;
4. implement conditional branches for each case;
5. validate input values;
6. implement repetition using loops;
7. test the program with multiple scenarios;
8. document and explain the solution.




### Expected Evidence

The task produces three main types of evidence:

1. **Source code**, implementing logic, conditions, and repetition;
2. **Documented reasoning**, including explanation of logic and validation;
3. **Test cases**, showing correct behavior across different input scenarios.



### Acquisition Context

The task is situated in an **introductory programming context**, integrated with Physics, for students in the **1st year [TECHNICAL_PROGRAM]**.

The interdisciplinary nature supports conceptual understanding but does not change the computational focus of the task.
